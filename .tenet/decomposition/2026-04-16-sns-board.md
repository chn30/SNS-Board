## Job Details

### job-1: Project Setup + Prisma Schema
- **Type**: dev
- **Dependencies**: none
- **Deliverables**:
  - `npx create-next-app` with TypeScript, Tailwind, App Router
  - Prisma init with PostgreSQL
  - Full schema (User, Post, Comment, Like, Report, AdminLog)
  - `prisma migrate dev`
  - Seed script with test users (test-user-1~5, test-admin-1)
  - ESLint + Prettier config
  - Tailwind config with Glassmorphism Neon design tokens
  - Project structure: src/app/, src/components/, src/lib/, src/actions/
- **Verification**: `npx prisma validate && npm run build` passes

### job-2: Auth (NextAuth + Mock SSO + Middleware)
- **Type**: dev
- **Dependencies**: job-1
- **Deliverables**:
  - NextAuth v5 setup with CredentialsProvider (mock SSO)
  - JWT strategy with userId + role only
  - middleware.ts: redirect unauthenticated to /login
  - /login page with SSO login button
  - Auth utilities: getCurrentUser(), requireAuth(), requireAdmin()
  - Session type augmentation (userId, role)
- **Verification**: Login with test-user-1, session persists, /login redirects authenticated users to /
- **Interface**: `getCurrentUser()` returns `{id, role}` or null

### job-3: Post CRUD (Server Actions + Service Layer)
- **Type**: dev
- **Dependencies**: job-2
- **Deliverables**:
  - src/lib/services/post.service.ts (DB queries)
  - src/actions/post.actions.ts (Server Actions with Zod validation)
  - getPosts (cursor pagination, category filter, sort)
  - getPost (single post + view count)
  - createPost
  - deletePost (soft delete, author or admin)
  - Popular feed algorithm implementation
  - Comment service + actions (getComments, createComment, deleteComment)
- **Verification**: Unit tests for service layer, Server Actions callable from test
- **Interface**: Returns sanitized data (no userId/email/ssoId)

### job-4: Feed Page (Main UI + Infinite Scroll)
- **Type**: dev
- **Dependencies**: job-3
- **Deliverables**:
  - src/app/(main)/page.tsx — Main feed page
  - PostCard component (Glassmorphism Neon style from DESIGN.md)
  - Tab switching (최신/인기)
  - Category filter pills
  - Infinite scroll with cursor pagination
  - "새 글 작성" button → /write or modal
  - Write post page/modal with category select + title + content
  - Left sidebar (categories, shortcuts)
  - Right sidebar (trending, stats)
  - 4-column layout from DESIGN.md
  - Responsive breakpoints
- **Verification**: Feed renders posts, tabs switch, categories filter, new post appears

### job-5: Post Detail Page + Comments
- **Type**: dev
- **Dependencies**: job-4
- **Deliverables**:
  - src/app/(main)/post/[id]/page.tsx
  - Full post display with category badge, HOT tag
  - Comment list
  - Comment input + submit
  - Back to feed navigation
  - Delete button (for own posts)
- **Verification**: Navigate to detail, see comments, add comment, comment count increases

### job-6: Like + Report (Toggle + Auto-hide)
- **Type**: dev
- **Dependencies**: job-5
- **Deliverables**:
  - Like toggle Server Action + optimistic UI update
  - Like button component (heart, active state)
  - Report modal (reason selection)
  - Report Server Action + duplicate check
  - Auto-hide logic (3 unique reports → isHidden=true)
  - Toast notifications
- **Verification**: Like toggles, count changes, report submits, 3rd report hides post

### e2e-1: Integration Test (User Flows)
- **Type**: integration_test
- **Dependencies**: job-6
- **Prompt**: Run acceptance tests for user-facing features. Start the dev server, run tests/acceptance/sns-board.spec.ts scenarios S1-S5. Report which tests pass and fail.

### job-7: Admin Pages (Dashboard + Post/Report Management)
- **Type**: dev
- **Dependencies**: job-6
- **Deliverables**:
  - src/app/(admin)/admin/page.tsx — Admin dashboard
  - src/app/(admin)/admin/posts/page.tsx — Post management (list, delete, restore)
  - src/app/(admin)/admin/reports/page.tsx — Report management (list, dismiss)
  - Admin layout with role check
  - AdminLog recording for all actions
  - Admin Server Actions (adminDeletePost, adminRestorePost, adminGetReports, adminDismissReport)
- **Verification**: Login as admin, see all posts including hidden, delete/restore works, logs recorded

### e2e-2: Final E2E (All Acceptance Tests)
- **Type**: integration_test
- **Dependencies**: job-7, e2e-1
- **Prompt**: Run ALL acceptance tests including admin scenarios. Start dev server, run full Playwright suite. Report all test results.

## Interface Contracts

### job-1 → job-2
- Prisma Client configured and connected
- User model with id, ssoId, email, role fields
- Seed data: test users exist in DB

### job-2 → job-3
- `getCurrentUser()` returns `{id: string, role: 'USER'|'ADMIN'}` or redirects
- `requireAuth()` throws/redirects if not authenticated
- `requireAdmin()` throws/redirects if not admin

### job-3 → job-4
- `getPosts({cursor?, category?, sort})` returns `{posts: Post[], nextCursor?}`
- `createPost({title, content, category})` returns created post
- Post object has: id, title, content, category, likeCount, commentCount, viewCount, createdAt, isLiked (boolean for current user)

### job-4 → job-5
- PostCard component exists and is reusable
- Navigation pattern from feed to /post/[id] established

### job-5 → job-6
- Like/Report button slots exist in PostCard and detail page
- Comment component has like button slot
