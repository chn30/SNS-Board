### Decisions Made (from user's initial requirements + interview)

- **Tech Stack**: Next.js + TypeScript + Tailwind CSS + Prisma + PostgreSQL
- **Auth**: 사내 SSO 연동 (MVP에서는 NextAuth.js + mock SSO, 추후 실제 SSO 연결)
- **Anonymity**: 화면에 "익명"으로만 표시, DB에는 userId 저장하여 내부 추적 가능
- **Categories**: 자유/질문/정보 3개 카테고리
- **Admin**: 별도 관리자 페이지 — 게시글/댓글 삭제, 신고 관리, 사용자 관리
- **Report System**: 신고 접수 + 누적 3회 시 자동 비공개 처리
- **Feed**: 최신글/인기글 탭 전환
- **Like**: 게시글/댓글 공감 (중복 방지)
- **Responsive**: 모바일/PC 반응형
- **MVP Focus**: 텍스트 중심, 이미지 업로드는 후순위
- **Scale**: 50-200명 동시 사용자
- **Deployment**: 미정 (Docker ready로 준비)

### MVP Acceptance Criteria

1. 피드 화면에서 게시글 목록 확인 가능 (최신/인기 탭)
2. 게시글 작성 가능 (카테고리 선택 포함)
3. 게시글 상세 진입 가능
4. 댓글 작성 가능
5. 공감 (좋아요) 가능 (게시글/댓글)
6. 신고 기능 (신고 접수 + 자동 비공개)
7. 관리자 페이지 (게시글/댓글 삭제, 신고 목록 확인)
8. 기본 Playwright e2e 테스트 포함
9. 모바일/PC 반응형 UI

## Round 3 (Final Gap Resolution — Agent Defaults)

### Decisions Made (filling all remaining gaps)

- **인기글 가중치 공식**: `score = likeCount / (hoursAge + 2)^1.5` — 시간 감쇠 함수, 2시간 오프셋으로 신규 글 보정
- **성능 목표**: 피드 로딩 <1초, 게시글 작성 응답 <500ms, 댓글 작성 응답 <500ms
- **부하 테스트**: MVP에서는 미포함, 추후 k6 등으로 200명 동시 접속 테스트
- **SSO 실패 시**: 로그인 페이지로 리다이렉트 + 에러 메시지 표시, fallback 없음
- **신고 비공개 알림**: 작성자에게 별도 알림 없음 (MVP). 추후 알림 시스템 추가 가능
- **관리자 지정 권한**: 최초 관리자는 DB seed로 생성. 관리자가 다른 관리자를 지정 가능
- **삭제 방식**: Soft delete (isDeleted flag). 게시글 삭제 시 댓글은 유지되나 "삭제된 게시글"로 표시
- **무한스크롤 페이지 크기**: 20개씩 로딩
- **공감 취소**: 가능 (토글 방식)
- **데이터 보존**: MVP에서는 별도 백업 정책 없음. PostgreSQL 기본 운영
- **접근성**: 기본 시맨틱 HTML + aria-label 적용. WCAG 2.1 AA 수준은 후순위

## Round 4 (Final micro-gap fill)

### Decisions Made

- **hoursAge 기준**: 게시글 작성 시점(createdAt) 기준
- **피드 1초 기준**: 사내 네트워크 환경, 게시글 1만 건 이하 기준. 브라우저 DOMContentLoaded 기준
- **SSO 전환 마이그레이션**: NextAuth provider만 교체하면 되도록 설계. userId는 SSO sub claim 매핑
- **DB 커넥션 풀**: Prisma 기본 connection_limit=10, 200명 규모에 충분. 추후 PgBouncer 도입 가능
- **신고 3회 기준**: 고유 사용자 기준 (같은 사용자 중복 신고 불가)
- **삭제 후 공감/신고**: soft delete이므로 카운트 유지. 피드에서만 숨김
- **감사 로그**: MVP에서는 관리자 삭제 시 간단한 AdminLog 테이블에 기록
- **403 리다이렉트**: 로그인 페이지로 리다이렉트
- **검색 기능**: MVP에서는 미포함. 후순위 기능

## Summary

회사 본부 구성원(50-200명)이 사용하는 익명 SNS형 게시판. 사내 SSO로 인증하되 화면에는 완전 익명으로 표시. Next.js + TypeScript + Tailwind + Prisma + PostgreSQL 스택. 게시글 작성/조회/삭제, 댓글, 공감, 신고(자동 비공개), 관리자 페이지를 MVP로 구현. 모바일/PC 반응형 SNS 피드 UI.
