# Scenarios: 사내 익명 SNS 게시판 (sns-board)

Date: 2026-04-16
Feature: sns-board

## Scenarios (Success)

### S1: 피드 조회
사용자가 SSO로 로그인한 후 메인 피드 페이지에 진입한다. 최신 게시글 20개가 무한스크롤로 로드된다. "최신" 탭과 "인기" 탭을 전환하면 정렬이 바뀐다. 카테고리 필터(전체/자유/질문/정보)를 선택하면 해당 카테고리 글만 표시된다.

### S2: 게시글 작성
사용자가 "새 글 작성" 버튼을 클릭하면 모달이 열린다. 카테고리를 선택하고 제목/내용을 입력한 뒤 "게시하기"를 클릭하면 게시글이 등록되고 피드 최상단에 표시된다. 화면에는 "익명"으로만 표시된다. DB에는 사용자 ID가 저장된다.

### S3: 게시글 상세 + 댓글
피드에서 게시글을 클릭하면 상세 페이지로 진입한다. 전체 본문, 공감 수, 댓글 목록이 표시된다. 댓글 입력란에 텍스트를 입력하고 "등록"을 누르면 댓글이 추가되고 댓글 수가 증가한다.

### S4: 공감 토글
사용자가 게시글/댓글의 하트 버튼을 클릭하면 공감 수가 1 증가하고 버튼이 활성화된다. 다시 클릭하면 공감이 취소되고 수가 1 감소한다. 같은 사용자가 같은 대상에 중복 공감은 불가능하다.

### S5: 신고 접수 및 자동 비공개
사용자가 신고 버튼을 클릭하면 신고 사유 선택 모달이 나타난다. 사유를 선택하고 제출하면 "신고가 접수되었습니다" 토스트가 표시된다. 같은 사용자가 같은 대상을 중복 신고할 수 없다. 서로 다른 3명의 사용자가 같은 게시글을 신고하면 해당 게시글이 자동으로 비공개(hidden) 처리되어 피드에서 사라진다.

### S6: 관리자 게시글/댓글 삭제
관리자가 관리자 페이지에서 게시글 목록을 확인한다. 특정 게시글의 삭제 버튼을 클릭하면 soft delete 처리되어 피드에서 사라진다. AdminLog에 삭제 기록이 남는다. 비공개 처리된 게시글을 복원할 수도 있다.

### S7: 관리자 신고 관리
관리자가 신고 목록 페이지에서 신고 내역을 확인한다. 각 신고의 사유와 횟수를 볼 수 있다. 관리자가 게시글을 삭제하거나 신고를 기각할 수 있다.

### S8: 인기글 가중치 정렬
인기 탭에서는 `score = likeCount / (hoursAge + 2)^1.5` 공식으로 계산된 점수 순서대로 게시글이 표시된다. 최근 공감이 많은 글이 상단에 위치한다.

## Anti-Scenarios (Failure)

### A1: 익명성 노출
어떤 상황에서도 사용자의 실명, 이메일, SSO ID가 프론트엔드에 노출되어서는 안 된다. API 응답에 userId, email, ssoId가 포함되면 안 된다.

### A2: 중복 공감/신고
같은 사용자가 같은 게시글/댓글에 여러 번 공감하여 공감 수를 조작하거나, 같은 대상을 중복 신고하여 부당하게 비공개 처리를 유발해서는 안 된다.

### A3: 비인가 관리자 접근
일반 사용자가 관리자 페이지 URL에 직접 접근하거나 관리자 API를 호출하여 게시글을 삭제하거나 사용자를 관리해서는 안 된다. 403 응답이 반환되어야 한다.

### A4: 미인증 접근
SSO 로그인하지 않은 사용자가 피드, 게시글, 댓글 등 어떤 기능에도 접근할 수 없어야 한다. 로그인 페이지로 리다이렉트되어야 한다.

### A5: XSS/인젝션 공격
게시글이나 댓글에 `<script>` 태그나 악성 HTML을 삽입하여 다른 사용자의 브라우저에서 실행되어서는 안 된다. 모든 사용자 입력은 이스케이프 처리되어야 한다.

### A6: 삭제된 콘텐츠 접근
soft delete된 게시글/댓글이 피드나 검색에서 표시되어서는 안 된다. API로 직접 조회해도 삭제된 콘텐츠임을 나타내는 응답만 반환되어야 한다.

### A7: 서버 오류 시 데이터 손실
게시글 작성/댓글 작성 중 서버 오류가 발생하면, 에러 메시지가 표시되고 사용자 입력이 보존되어야 한다. 부분적으로 저장된 데이터가 남아서는 안 된다.
