# Spec: 사내 익명 SNS 게시판 (sns-board)

Date: 2026-04-16
Feature: sns-board

## Purpose
회사 본부 구성원(50-200명)이 사용하는 익명 SNS형 게시판. 사내 SSO로 인증하되 화면에는 완전 익명으로 표시. 게시글/댓글/공감/신고 기능과 관리자 운영 도구를 제공하는 MVP.

## Tech Stack
- **Runtime**: Node.js 20+
- **Framework**: Next.js 15 (App Router)
- **Language**: TypeScript 5.x (strict mode)
- **Styling**: Tailwind CSS 3.x
- **ORM**: Prisma 5.x
- **Database**: PostgreSQL 16+
- **Auth**: NextAuth.js v5 (Auth.js) with JWT strategy
- **Validation**: Zod
- **Testing**: Playwright (e2e), Vitest (unit)
- **Linting**: ESLint + Prettier

## API (Server Actions)

### Posts
| Action | Input | Auth | Description |
|--------|-------|------|-------------|
| getPosts | cursor?, category?, sort(latest/popular) | user | 피드 조회 (20개씩 cursor pagination) |
| getPost | postId | user | 게시글 상세 조회 + viewCount 증가 |
| createPost | title, content, category | user | 게시글 생성 |
| deletePost | postId | author or admin | 게시글 삭제 (soft delete) |

### Comments
| Action | Input | Auth | Description |
|--------|-------|------|-------------|
| getComments | postId, cursor? | user | 댓글 목록 조회 |
| createComment | postId, content | user | 댓글 생성 |

### Likes
| Action | Input | Auth | Description |
|--------|-------|------|-------------|

### Reports
| Action | Input | Auth | Description |
|--------|-------|------|-------------|

### Admin
| Action | Input | Auth | Description |
|--------|-------|------|-------------|

## Auth Flow
1. 사용자가 사이트에 접근한다.
2. middleware.ts가 세션을 확인한다 (JWT).
3. 세션이 없으면 /login으로 리다이렉트.
4. /login에서 SSO 버튼을 클릭하면 NextAuth signIn 흐름 시작.
5. MVP에서는 CredentialsProvider로 mock SSO 구현 (test-user-1~5, test-admin-1).
6. 인증 성공 시 JWT 발급, 피드 페이지로 리다이렉트.
7. JWT에는 userId, role만 포함 (email, ssoId 미포함 — 프론트 노출 방지).

## Page Routes
- `/login` — 로그인 페이지
- `/` — 메인 피드 (최신/인기 탭, 카테고리 필터)
- `/post/[id]` — 게시글 상세
- `/write` — 게시글 작성 (또는 모달)
- `/admin` — 관리자 대시보드
- `/admin/posts` — 관리자 게시글 관리
- `/admin/reports` — 관리자 신고 관리

## Success Criteria
1. 로그인 후 메인 피드에서 게시글 20개가 1초 이내 로딩된다.
2. 게시글 작성 후 피드 최상단에 즉시 표시된다.
3. 게시글 상세에서 댓글을 작성하면 댓글 목록에 즉시 추가된다.
4. 공감 버튼 클릭 시 즉각 수치가 변동된다 (optimistic update).
5. 신고 3회 누적 시 해당 게시글이 피드에서 자동으로 사라진다.
6. 관리자 페이지에서 게시글 삭제/복원이 가능하다.
7. API 응답에 userId, email, ssoId가 절대 포함되지 않는다.
8. 비인증 사용자는 모든 페이지에서 /login으로 리다이렉트된다.
9. 일반 사용자는 /admin 접근 시 403 또는 리다이렉트된다.
10. Playwright e2e 테스트가 모든 핵심 시나리오를 커버한다.

## Popular Feed Algorithm
```
score = likeCount / (hoursAge + 2) ^ 1.5
```
- `hoursAge` = (now - createdAt) in hours
- 공감 수 0인 게시글도 표시하되 최하단 정렬
